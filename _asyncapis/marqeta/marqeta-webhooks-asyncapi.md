description: Marqeta delivers real-time event notifications to a developer-configured HTTPS endpoint when specific events occur within a card program. Each program supports up to five active webhook configurations. Events cover card transactions, authorizations, declines, card state transitions, user state transitions, and other platform activity. Webhook payloads are delivered as HTTP POST requests in JSON format to the receiving endpoint. The Marqeta platform includes Basic Auth credentials in the notification request using the username and password configured in the webhook setup. Event types can be subscribed to individually (e.g., transaction.authorization) or as a group using a wildcard (e.g., transaction.* or cardtransition.*). Applications should respond to webhook deliveries with an HTTP 200 status within 10 seconds; failures trigger automatic retries.
