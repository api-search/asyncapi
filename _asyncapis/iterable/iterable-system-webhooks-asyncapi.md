description: Iterable system webhooks send real-time event data from an Iterable project to external systems via HTTP POST requests whenever specified events occur. System webhooks can be configured to fire on email sends, opens, clicks, bounces, unsubscribes, complaints, push notifications, SMS messages, in-app messages, and other engagement events. Webhook payloads are sent as JSON to a user-configured HTTPS endpoint and support no authentication, HTTP Basic authentication, or OAuth 2.0.
